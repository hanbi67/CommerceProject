# CH2. 커머스 과제

### 프로젝트 개요
- 프로젝트 목적: 객체지향 프로그래밍과 계층적 구조 설계 학습을 위한 커머스 시스템 구현
- 주요 학습 목표: 객체 지향 설계 구조를 커머스 플랫폼에 적용시켜 기본적인 커머스 플랫폼의 핵심 기능 구현


- 실시간 커머스 플랫폼(콘솔) 구현
- 사용자: 카테고리 조회 → 상품 조회 → 장바구니 담기 → 주문/취소
- 관리자: 비밀번호 인증 → 상품 추가/수정/삭제/전체 상품 조회

### 기술 스택
- Language: Java 17
- IDE: IntelliJ IDEA

___

### 수행한 단계

`필수 기능`

 - [STEP 1] : 객체 지향 설계를 적용한 상품 관리 시스템 프로그래밍
 - [STEP 2] : 객체 지향 설계를 적용해 순서 제어를 클래스로 관리
 - [STEP 3] : 상품 카테고리와 고객 관리 클래스 기반으로 관리
 - [STEP 4] : 캡슐화 적용

`도전 기능` [Lv 1] : 장바구니 및 주문하기 기능 추가
  - 장바구니 생성 및 관리 기능
  - 재고 관리 시스템
  - 장바구니 출력 및 금액 계산
  - 장바구니 담기 기능
  - 주문 기능

`도전 기능` [Lv 2] : 관리자 모드 추가
 - 관리자 인증 기능
 - 상품 추가 기능
 - 상품 수정 기능
 - 상품 삭제 기능
 - 전체 상품 현황
---
### 기능 목록
`사용자 기능`
- 카테고리 목록 출력 
- 상품 목록 출력 
- 상품 상세 확인 
- 장바구니 담기(수량 입력)
- 주문 관리(장바구니 확인/주문 확정/주문 취소)
- 재고 차감

`관리자 기능`
- 관리자 비밀번호 설정/인증
- 상품 추가 (상품명 중복 검증)
- 상품 수정 (상품명 검색 후 가격/설명/재고 수정)
- 상품 삭제 (삭제 시 장바구니에서도 제거)
- 전체 상품 현황 출력

### 📂 클래스 구조
```
CommerceProject
└─ src
   ├─ Main.java                # 프로그램 시작점 (CommerceSystem.start() 호출)
   ├─ CommerceSystem.java      # 메인 메뉴/사용자 흐름
   ├─ AdminModeSystem.java     # 관리자 인증/관리자 메뉴    
   ├─ Category.java            # 카테고리 정보 + 상품 리스트 보관
   ├─ Product.java             # 상품 정보(이름/가격/설명/재고)
   ├─ ShoppingCart.java        # 장바구니(Map<Product, 수량>)
   │                             주문 처리(총액/재고 차감)
   └─ Customer.java            # 고객 관련 기능(미사용)
```

### 실행 방법
- Main 클래스 main 메서드 실행

### 메뉴 흐름
1. 메인 → 카테고리 선택 → 상품 선택 → 장바구니 담기
2. 메인 → 주문 관리 → 주문 확정(재고 차감)
3. 메인 → 관리자 모드 → 상품 추가/수정/삭제/전체 현황

### 주요 클래스 · 역할 설명
- Main: 프로그램 시작점
- CommerceSystem: 메인 메뉴/입력 흐름 제어, 주문 관리 진입
- AdminModeSystem: 관리자 인증, 관리자 메뉴, 상품 CRUD
- Category: 카테고리명 + 상품 리스트 보유
- Product: 상품 데이터 + 출력 메서드
- ShoppingCart: Map<Product, Integer> 기반 장바구니, 주문 금액 계산, 주문 확정(재고 차감)
